ARP(Address Resolution Protocol)

개요

프레임을 형성할 때는 Destination MAC address가 필요합니다. 그래서 (목적지) MAC 주소를 모를 때 (목적지) IP 주소를 통해 (목적지) MAC 주소를 알아낼 때 사용하는 프로토콜입니다.

*다른 네트워크(WAN)와 통신할 때 IP는 상대방의 IP를 그대로 사용하지만, MAC은 상대방도 Gateway 라우터의 MAC을 사용합니다. 즉 자신의 LAN을 넘어가는 ARP 요청에서는 상대쪽 Gateway의 MAC이 응답합니다.
*remote lan segment와 통신할 경우, 즉 원격 접속 PC가 목적지라면 MAC 주소를 찾을 수 없습니다. 그래서 이런 경우 Destination IP의 Gateway 라우터 MAC을 사용합니다.

설명

자신의 IP와 상대방의 IP주소를 담아서 MAC Broadcast(FF:FF:FF:FF:FF:FF)로 LAN 전체에 요청(ARP Request 패킷 전송)을 보내면 해당 IP의 상대방이 자신의 MAC주소와 함께 응답(ARP Reply 패킷 전송)을 보냅니다.

*RARP (Reverse ARP)

GARP (Gratutious ARP)

:LAN 전체에 자신의 IP와 MAC을 Broadcast로 보내어 ARP Cache를 갱신하도록 알려주는 서비스입니다.
