Source Port Address:송신지의 어떤 프로세스가 주는지에 대한 주소를 담고 있는 영역입니다.

Destination Port Address: 수신지의 어떤 프로세스가 받을지에 대한 주소를 담고 있는 영역입니다.


Sequence Number: 형성되는 세그먼트마다 MSS 단위로 순서가 되는 번호를 지정받고 저장하는 영역입니다. (MSS 단위의 첫바이트가 Sequence Number가 되며, 순서는 임의의 숫자로 시작하여 최대값에 도달하면 0부터 시작합니다.)


Acknowledgement Number: 수신자가 다음에 받아야할 Sequence Number를 확인 응답으로 보내는 영역입니다. (수신자가 이전 세그먼트를 제대로 받지 못했다면 이전 세그먼트의 Sequence Number를 보내고 이전 세그먼트를 제대로 받았다면 예상되는 다음 세그먼트의 Sequence Number를 보냅니다.)


HLEN: OPtions & Padding영역으로 인해 유동적으로 변하는 헤더의 길이를 나타내는 영역입니다.

Reserved: 미래에 사용하기 위해 남겨둔 예약 영역으로 현재는 사용되지 않습니다.(값이 0으로 채워집니다.)

Control Flags

-URG(URGent)
: 송신측 상위 계층에서 긴급 데이터임을 알려주면 Urgent Pointer에 값을 채우고 곧 해당 세그먼트를 순서에 상관없이 먼저 전송합니다.
-ACK(ACKnowledgment)
: 값이 채워져 있으면 Acknowledgement number 값이 준비되었음을 알립니다.(SYN 세그먼트 전송 이후 항상 값이 채워져 있습니다.)
-PSH(PuSH)
: 값이 채워져 있으면 수신 받은 버퍼링 데이터를 가능한 한 빨리 Application Layer에 전달할 것을 알립니다.(강제 전송)

-RST(ReSeT)
: 연결 상의 문제가 발견되었을 때 강제로 초기화(연결 종료) 시키는 용도로 사용됩니다.
-SYN(SYNcronize)
: 연결을 시작할 때 Sequence Number 동기화에 사용됩니다.
-FIN(FINish)
: 연결을 종료하고 싶다는 뜻을 상대방에게 전달할 때 이 값을 채워서 보냅니다.

Window Size: 수신측에서 자신의 버퍼 여유공간을 지속적으로 통보하여 송신측에서 이에 맞는 크기로 세그먼트를 보내줄 것을 알리는데 사용됩니다.


Checksum: 오류를 검사하기 위한 체크섬(일정 방식으로 헤더의 데이터를 더한 값)을 저장하는 영역입니다. (송신자가 세그먼트의 체크섬을 함께 보내면 수신자는 송신자의 체크섬과 자신이 받은 체크섬을 비교하여 오류를 검사합니다.)

Urgent Pointer: URG가 설정되었을 때 긴급 데이터의 위치가 Sequence Number로부터 어디까지인지 표현해주는데 사용되는 영역입니다.

연결 설정 -3Ways Handshake

Segment1(연결 요청)
-A측 Sequence Number(a) 소비
-SYN 값 '1', ACK 값 '0' 설정
-(Data X, Options에 MSS 포함)

Segment2(연결 수락)
-B측 Sequence Number(b) 소비
-SYN, ACK 값 '1' 설정
-Acknowledgment Number(a+1) 소비
-(Data X, Options에 MSS 포함)

Segment3(연결 설정)
-A측 Sequence Number(a+1) 사용
-SYN 값 '0', ACK 값 '1'설정
Acknowledgment Number(_b+1) 소비
-(Data X, Options 사용 안함)

연결 해제 -4Ways Handshake

Segment1(A측 종료 요청)
-A측 Sequence Number(x) 소비
-FIN 값 '1', ACK 값 '0' 설정

Segment2(A측 종료 확인)
-B측 Sequence Number(y) 소비
-FIN, ACK 값 '1' 설정
-Acknowledgment Number(x+1) 소비

Segment3(B측 종료 요청)
-B측 Sequence Number(y) 사용
-FIN, ACK 값 '1' 설정
-Acknowledgment Number(x+1) 사용

Segment4(B측 종료 확인)
-A측 Sequence Number(x) 사용
-FIN 값 '0', ACK 값 '1' 설정
-Acknowledgment Number(y+1) 소비

흐름 제어 -Sliding Window

전송되는 데이터의 양을 조절합니다.
수신측에서 ACK과 함께 Window Size를 수시로 알려주면 송신측에서는 이에 맞는 Window Size 정하여 전송을 시작합니다. Window에 포함되는 Segment들은 ACK을 받지 않아도 전송을 계속합니다. ACK을 받은 Segment들은 Window에서 제외되며 그 크기만큼 다음 Segment를 Window에 포함시키게 됩니다. 이로써 송신지와 수신지 사이의 흐름을 제어하는 역할을 수행합니다.

오류 제어 -Go Back n / Selective Repeat

전송 도중 데이터가 유실되거나 잘못된 데이터가 수신되었을 경우에 대처합니다. TCP 전송에서는 타이머가 작동합니다. 송신측에서는 일정 시간 이내에 수신측의 확인 응답이 없을 경우 이를 오류로 간주하고 재전송합니다.
이때 오류가 발생한 지점부터 재전송을 시작하면 Go Back n 기법인 것이고, 오류가 발생한 부분만 재전송하면 Selective Repeat 기법이 된다.

세부목록

*MSS(MAX Segment Size)

:하나의 세그먼트가 포함할 수 있는 바이트 단위의 최대 크기를 말합니다.

*Segmetation

:상위 계층에서 온 데이터를 전송 계층에서 받아들일 수 있는 크기로 나누는 과정을 말합니다.
