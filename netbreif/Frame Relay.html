Frame Relay

개요

ISP 회선에 접속하여 WAN에 참여하고자 할 때 패킷 스위칭 방식을 사용할 경우 라우터에서 해당 프로토콜을 설정합니다. 이더넷과의 차이점은 브로드캐스트 방식을 사용하지 않고 virtual circuit(가상 인터페이스)을 통해서만 통신한다는 점입니다.
프레임 릴레이의 encapsulation 방식은 IETF의 표준화 방식과 cisco만의 비표준화 방식이 있는데 cisco 장비들끼리 통신할 경우 cisco 방식을 default로 사용하고 타사 장비와의 통신시 IETF 설정을 합니다.

DLCI(Data-Link Connection Identifier)

DLCI라는 내부 식별자를 사용합니다. 하나의 물리적 회선을 여러 개의 논리적 회선으로 나누어 사용하는데 각각의 논리 회선(채널)에 DLCI를 부여합니다.

LMI

LMI는 인터페이스의 동작 상태, DLCI 정보등을 담아 알려주는 역할을 하는데 타입이 다양해서(Cisco , ansi, q993a) 통신하고자 하는 ISP와 버전을 통일해야합니다. 요즘 라우터들은 알아서 맞추기 때문에 따로 설정해줄 필요는 없습니다.

동작 과정

1.피지컬인터페이스(서브인터페이스의 원래 인터페이스) 설정
LMI 타입 설정, encapsulation 방식 설정

2.서브인터페이스 생성 및 설정
인터페이스 타입 설정, DLCI 연결 설정, IP 설정

*DLCI 수동 연결 설정
P2P의 경우:frame-relay interface-dlci [Local DLCI Number]
그외(Multi-point...)의 경우:frame-relay map [protocol] [Address to map] [Local DLCI Number] broadcast

*프레임 릴레이 스위치 설정
스위치의 역할을 해서 스위치라고 하지만 라우터에 설정해주는 기능입니다. frame-relay switching 이라고 설정하면 해당 라우터는 프레임 릴레이 스위치의 역할을 수행합니다. 해당 라우터의 인터페이스에는 ip를 부여할 필요없이 frame-relay route [Source DLCI Number] interface [port/slot] [Destination DLCI Number]만 설정합니다.
