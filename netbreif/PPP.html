PPP(Point to Point Protocol)

개요

ISP 회선에 접속하여 WAN에 참여하고자 할 때 전용선 방식이나 회선 스위칭 방식을 사용할 경우 라우터에서 해당 프로토콜을 설정합니다. 2계층 프레임의 encapsulation 방식을 규정하는 프로토콜이며 같은 계열의 이더넷, ATM, HDLC등에는 없는 보안 기능을 가지고 있습니다.
3계층의 여러 프로토콜(IP,IPX,Appletalk...)을 통합하여 encapsulation할 수 있습니다.
NCP(Network Control Protocol)로 멀티 프로토콜을 지원합니다.
LCP(Link Control Protocol)로 보안 기능을 지원힙니다.
데이터 링크 계층의 세션을 구축한 직후 보안인증 단계를 거쳐 네트워크 계층 세션을 구축하며, 주로 CHAP(Challenge Handshake Authentication Protocol)로 보안인증을 합니다.

HDLC(High-level Data Link Protocol)

:PPP와 마찬가지로 2계층 encapsulation에 사용되는 프로토콜입니다. 표준 버전에서 encapsulation을 지원하는 3계층 프로토콜이 ip밖에 없어 잘 쓰이지 않습니다.

SLIP(Serial Line Internet Protocol)

:PPP와 마찬가지로 2계층 encapsulation용 비표준 프로토콜입니다. 구조가 단순하지만 보안 기능이 없습니다. PPP 등장 이후 잘 쓰이지 않습니다.

PAP(Password Authentication Protocol)

:PPP에서 사용하는 간단한 보안인증 프로토콜입니다. host name과 P/W를 암호화없이 통신하여 인증하기 때문에 해킹에 취약해 잘 쓰이지 않습니다.
