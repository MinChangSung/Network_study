hot keys

Ctrl+C → cancels any command and returns directly to privileged EXEC mode

Ctrl+Z → returns to privileged EXEC mode

"exit" → returns to previous mode

tab → completes the remainder of keyword

? → command help of current mode

Ctrl+P → Recalls commands in the history buffer of past typing command

Ctrl+N(ctrl+p쓰다가 앞으로 넘겨보고싶을 때)

show command

#show startup-config(=show config)
:NVRAM에 저장된 백업 구성 파일 정보 확인

#show running-config
:RAM에서 현재 사용중인 구성 파일 정보 확인

#show version
:라우터의 기본 정보 확인

#show interface (+port number)
:(해당or모든) 인터페이스의 현재 상태 확인

#show flash
:플래시 메모리 정보 확인

#show processes cpu
:라우터의 현재 동작 상태 확인

#show ip route

memory command

#write memory
:RAM에서 설정 중이던 구성을 NVRAM에 백업합니다.

#configure memory
:RAM에서 설정 중이던 구성을 전에 NVRAM에 백업된 내용으로 덮어씁니다.

#erase startup-config
:NVRAM에 저장된 구성 파일을 지워버립니다.

password command

(config)#enable password [password]
:privileged mode 접속에 필요한 암호를 설정합니다. show command로 암호를 확인할 수 있습니다. secret command보다 우선순위가 낮아 secret 설정이 없는 경우만 비밀번호로 사용됩니다.

(config)#enable secret [password]
:privileged mode 접속에 필요한 암호를 설정합니다. 자체 암호화가 한 번 더 이루어져 show command로도 암호를 알 수 없습니다. (잘 사용하지 않는 명령어입니다.)

(config)#line vty [minimum] [maximum]
(config-line)#login
(config-line)#password [password]
:telnet 접속에 필요한 암호를 설정합니다.

(config)#line con 0
(config-line)#login
(config-line)#password [password]
:console 접속에 필요한 암호를 설정합니다.

(config)#line aux 0
(config-line)#login
(config-line)#password [password]
:aux 접속에 필요한 암호를 설정합니다.

NVRAM

:RAM은 전원이 꺼지면 초기화되기 때문에 저장이 필요한 구성 파일의 내용을 NVRAM에 저장합니다.

Flash Memory

:NVRAM은 구성파일만 저장할 정도의 용량을 가집니다. IOS의 이미지 파일같이 용량이 큰 파일을 저장하는데는 플래시 메모리를 사용합니다.
