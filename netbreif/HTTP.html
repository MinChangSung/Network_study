HTTP(HyperText Transfer Protocol) (port:80)

소개

1989년 Tim Berners-Lee가 전세계가 연결된 통신망이라는 WWW(world Wide Web)의 개념을 고안하며 이를 위해 만든 프로토콜입니다.

이진수가 아닌 아스키 코드를 기본 전송단위로 하여 프로토콜 포맷(규격)에서 비트 길이가 아닌 공백 문자, 개행 문자 등으로 메시지 단위를 구분합니다.

기본적으로 TCP 연결 방식을 사용하지만 전송이 끝나면 연결을 유지하지 않는 비연결식 방식을 사용합니다.

HTTP Version

[http/1.0]: 한 번의 TCP 연결로 하나의 object만 전송하고 연결 종료, object가 많으면 여러 차례 TCP 연결을 반복
[http/1.1]: 한 번의 TCP 연결로 해당 페이지내 모든 object 전송 후 연결 종료
[http/2]: https
[http/3]: https에서 TCP가 아닌 구글에서 개발한 QUIC(Quick UDP Internet Connections) 프로토콜로 HTTP를 연결하며 스트리밍 서비스 등에 사용합니다.

request message format

method type sp Request-URL sp http version cr lf
header filed name: value cr lf
...
cr lf
+
body

[method type]
-GET: 클라이언트가 서버에게 URL 요청
-HEAD: GET 요청으로 반환될 데이터 중 헤더 데이터만 요청
-POST: 클라이언트가 서버에게 처리할 자료를 전송
-PUT: 클라이언트가 서버에게 지정한 URL에 지정한 데이터를 저장할 것을 요청
-PATCH: 클라이언트가 서버에게 지정한 URL 데이터를 부분적으로 수정할 것을 요청
-DELETE: 클라이언트가 서버에게 지정한 URL 정보를 제거해줄 것을 요청
-TRACE: 클라이언트가 서버에게 요청한 내용을 다시 반환해줄 것을 요청
-CONNECT: 클라이언트가 특정 프록시 서버에 연결 요청
-OPTIONS: 해당 URL에서 지원하는 요청 메세지 목록 요청

response message format

http version sp state code sp reason phrase cr lf
header filed name: value cr lf
...
cr lf
+
body

[state code]
-1xx: 임시응답, 현재까지 요청이 처리되었으니 계속 진행하라는 의미
-2xx: 성공
-3xx: 다시시도, 완전한 처리를 위해 추가적인 동작이 필요한 경우
-4xx: 클라이언트측 에러
-5xx: 서버측 에러

HTTPS(HyperText Transfer Protocol Secure) (port:443)

HTTP가 아스키 코드로 전송되어 더욱이 보안에 취약한 문제를 보완하기 위해 나온 HTTP의 보안 버전입니다. 암호화 관련 트래픽이 증가하고 인증서 유지 비용이 추가됩니다.

HTTPS에서 사용하는 주요 보안방식
[TLS(Transport Layer Security)]: SSL(Secure Socket Layer)이 표준화된 후속 버전, 지원 가능한 알고리즘을 서로 교환하고, 키 교환 및 인증, 암호화 단계를 거칩니다.
[VPN(Virtual Private Network)]: VPN 서버를 경유하여 인터넷에 접속시키는 보안방식, VPN에서 암호화하여 송수신합니다.

응답 코드 예시

-200: OK(성공)
-301: 요청한 URL이 완전 다른 주소로 이동했다는 의미, 서버는 어디로 이동했는지 위치를 함께 전송
-304: 변경 사항없음, 지정된 시각 이후 변경이 없으니 재접속하지 않고 클라이언트측에 캐시된 버전을 유지하겠다는 의미
