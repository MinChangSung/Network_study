DHCP(Dynamic Host Configuration Protocol)

동작과정

원래 전용 서버를 사용하여 구축하지만, 시스코 라우터의 경우 DHCP 서버 기능을 제공하는 경우가 많습니다. DHCP의 서버는 자신이 가지고 있는 주소 범위내에서 클라이언트에게 IP를 일정기간 임대합니다.

DHCP 서버 설정
1.자동 할당시 제외할 주소 범위 설정 (DNS server,DHCP server...)
- (config)#ip dhcp excluded-address [Start IP-Address] [End IP-Address]
2.할당할 주소 범위 설정(+default gateway 설정)
- (config)#ip dhcp pool [Pool Name]
- (dhcp-config)#network [N/W Address] [Subnet mask]
- (dhcp-config)#default-router [IP Address]
(- (dhcp-config)#dns-server [IP Address])

클라이언트 설정
- (config)#interface [Interface Type Slot/Port]

DHCP 메세지

1.Discover: 클라이언트가 서버에게 IP 관련 정보를 요청합니다.
2.Offer: 서버에서 클라이언트에게 IP 관련 정보를 전달합니다.
3.Request: 클라이언트가 서버에게 IP를 사용 의사를 밝힙니다.
4.ACK: 서버에서 클라이언트에게 IP 사용을 최종 허가합니다.
