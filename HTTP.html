HTTP(HyperText Transfer Protocol) (port:80)

소개

1989년 Tim Berners-Lee가 전세계가 연결된 통신망이라는 WWW(world Wide Web)의 개념을 고안하며 이를 위해 만든 프로토콜입니다.

이진수가 아닌 아스키 코드를 기본 전송단위로 하여 프로토콜 포맷(규격)에서 비트 길이가 아닌 공백 문자, 개행 문자 등으로 메시지 단위를 구분합니다.

기본적으로 TCP 연결 방식을 사용하지만 전송이 끝나면 연결을 유지하지 않는 비연결 방식을 사용합니다.

HTTP Version

[http/1.0]: 요청마다 새로운 TCP 연결
[http/1.1]: 같은 호소트에 대한 여러 요청은 한 번의 TCP 연결을 통해 처리
[http/2]: https
[http/3]: 구글에서 개발한 QUIC(Quick UDP Internet Connections) 프로토콜로 연결
-사용자 IP가 바뀌어도 연결을 유지
-Zero RTT(Round Trip Time): 최초 접속에서만 핸드쉐이크로 연결하고 이후에는 이전 통신기록을 토대로 연결하여 연결 설정 시간을 없습니다.
-TLS 보안
-패킷 손실에 대한 빠른 대응

[method type]
-GET: 클라이언트가 해당 URL로 데이터를 요청
-POST: 클라이언트가 해당 URL로 데이터를 전송
-HEAD: GET 요청으로 반환될 데이터 중 헤더 데이터만 요청
-PUT: 서버 자원 전체를 새로운 자원으로 교체하고자 할 때
-PATCH: 서버 자원에서 지정한 자원만 부분적으로 수정할 때
-DELETE: 서버 자원을 삭제할 때
-TRACE: 서버에 요청한 내용을 다시 반환해줄 것을 요청
-CONNECT: 특정 프록시 서버에 연결 요청
-OPTIONS: 해당 URL에서 지원하는 요청 메세지 목록 요청

[state code]
-1xx: 임시응답, 현재까지 요청이 처리되었으니 계속 진행하라는 의미

-5xx: 서버측 에러

HTTPS(HyperText Transfer Protocol Secure) (port:443)

HTTP가 아스키 코드로 전송되어 더욱이 보안에 취약한 문제를 보완하기 위해 나온 HTTP의 보안 버전입니다. 암호화 관련 트래픽이 증가하고 인증서 유지 비용이 추가됩니다.

<HTTPS 주요 보안방식>
[TLS(Transport Layer Security)]: SSL(Secure Socket Layer)이 표준화된 후속 버전, 지원 가능한 알고리즘을 서로 교환하고, 키 교환 및 인증, 암호화 단계를 거칩니다.
[VPN(Virtual Private Network)]: VPN 서버를 경유하여 인터넷에 접속시키는 보안방식, VPN에서 암호화하여 송수신합니다.

응답 코드 예시

-200: OK(성공)
-301: 요청한 URL이 완전 다른 주소로 이동했다는 의미, 서버는 어디로 이동했는지 위치를 함께 전송
-304: 변경 사항없음, 지정된 시각 이후 변경이 없으니 재접속하지 않고 클라이언트측에 캐시된 버전을 유지하겠다는 의미
