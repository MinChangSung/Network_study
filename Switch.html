Switching

(MAC table 관리)
Learning: 송신지 주소가 MAC 테이블에 없는 경우 해당 Port와 MAC 주소의 연관성을 MAC 테이블에 갱신합니다.
Flooding: 수신지 주소가 MAC 테이블에 없는 경우 모든 포트로 해당 프레임을 전송해버립니다.

(Collision domain 분리)
Filtering: 송신지와 수신지 주소가 같은 collision domain에 위치했을 때 다른 collision domain은 차단하고 전송합니다. (스위치의 collision domain은 각각의 port별로 나뉩니다.)
Forwarding: 송신지와 수신지 주소가 다른 collision domain 사이에 있을 때 두 포트만 연결합니다.
(스위치의 Filtering 기능으로 충돌 도메인을 분리함으로써 이더넷의 CSMA/CD로 인한 전송 대기 상태를 해소합니다.)

*Aging: MAC 테이블이 주소를 기억할 시간을 설정합니다. 기본값은 5분으로, 해당 시간내에 같은 주소로 통신이 이루어지지 않으면 MAC 테이블에서 삭제합니다.

VLAN(Virtual LANs)

스위치의 한 port에 VLAN을 설정하면 동일한 VLAN으로 설정된 port끼리만 브로드캐스트를 공유합니다. 즉 브로드캐스트 도메인을 IP(3계층 장비)를 사용하지 않고 2계층에서 분리할 수 있는 방식입니다. 다만 이렇게 VLAN으로 분리된 브로드캐스트 도메인은 라우팅이 되지 않으면 상호간의 통신을 할 수 없습니다. inter-VLAN, 즉 라우터의 subinterface를 통해 라우팅할 수 있습니다.

1.VLAN 생성
- #vlan database
- (vlan)#vlan[number]
 2.Port 연결(port based VLAN)
- (config)#interface [Interface Type Slot/Port]
- (config-if)#switchport access vlan[number]
 3.Trunk 설정
(Trunk protocol 설정으로 서로 다른 VLAN의 프레임들이 각자 자신의 VLAN 식별용 태그 헤더를 부착받습니다. 이로써 하나의 경로로 전송되는 여러 VLAN의 정보를 구분합니다.)
- (config)#interface [Inteface Type Slot/Port]
- (config-if)#switchport trunk encapsulation dot1q(자동설정 되어있는 경우가 많습니다.)
- (config-if)#switchport mode trunk
 *(VLAN 삭제)
- #vlan database
- (vlan)#no vlan[number]

SVI(switch Virtual Interface)
(= interface VLAN 1)

SNMP, Telnet등을 목적으로 스위치에서 IP를 할당할 수 있도록 제공되는 인터페이스입니다. (ip 설정 후 config모드로 나와 ip default-gateway 주소도 입력해야합니다.)

dot1q

VLAN의 encapsulation(tagging) 방식 중 하나입니다. IEEE802.1q와 ISL 두가지 방식이 있는데 ISL은 잘쓰이지 않고 IEEE802.1q를 명령어상에서는 dot1q로 표기합니다.

Stackable형

허브나 스위치 같은 장비를 연결하여 하나의 장비처럼 사용할 때 관련된 소프트웨어적 기능이 추가로 들어있는 장비를 Stackable 장비라고 합니다.
해당 기능이 없는 Standalone 장비는 하드웨어적 연결만 가능합니다.
